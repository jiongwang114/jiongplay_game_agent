# 待做 6 项功能

> 生成日期：2026-06-05 | 基于 Claude Code 分析

---

## 1. 设置弹窗 ↔ 后端联动

**现状**：弹窗选的预算/类型/平台只存 localStorage，`POST /chat` 不包含偏好信息

**方案**：扩展 `ChatRequest` 加 `settings` 字段，Agent 注入系统提示词 + 影响工具链参数

**涉及文件**：`server.py` `core/agent.py` `prompts/game_expert_prompt.txt` `frontend/app.js`

---

## 2. 真实用户认证

**现状**：无登录系统，"Player_One" 硬编码，头像固定

**方案**：新建 `data_layer/auth.py`，SQLite `users` 表，注册/登录 API，JWT token，前端登录弹窗

**涉及文件**：`data_layer/schema.py` `data_layer/auth.py`（新建）`server.py` `frontend/index.html` `frontend/app.js`

---

## 3. 多用户隔离

**现状**：`UserProfile` 是单例，所有 session 共享一个 `_profile` 字典

**方案**：`_profile` 改为 `dict[str, dict]`，按 `session_id`/`user_id` 隔离

**涉及文件**：`core/memory/user_profile.py` `core/agent.py`

---

## 4. 对话持久化

**现状**：`SessionMemory` 纯内存存储，重启丢失

**方案**：新增 `Conversation` SQLite 表，`SessionMemory` 升级为写入 DB + 内存缓存

**涉及文件**：`data_layer/schema.py` `core/memory/session_memory.py` `core/agent.py`

---

## 5. 图片上传

**现状**：📎 按钮仅弹 Toast

**方案**：`POST /api/upload`，保存到 `data/uploads/`，前端创建 `<input type="file">`，聊天中展示图片

**涉及文件**：`server.py` `frontend/app.js`

---

## 6. Agent 中间状态 SSE 推送

**现状**：前端 2s 轮询 `/api/agent-status`，有延迟

**方案**：新增 `GET /api/agent-stream/{session_id}` SSE 端点，基于 `asyncio.Queue` 实时推送

**涉及文件**：`server.py` `core/agent.py` `frontend/app.js`

---

## 实施顺序

1. 数据库表（User + Conversation）
2. 用户认证（auth.py + API + 前端）
3. 多用户隔离（UserProfile 重构）
4. 对话持久化（SessionMemory 升级）
5. 设置联动（ChatRequest + 提示词）
6. 图片上传（upload 端点 + 前端）
7. SSE 推送（agent-stream + 前端订阅）
